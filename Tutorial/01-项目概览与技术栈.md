# 01 · 项目概览与技术栈

## 1. 项目定位

CloudExplorer-Lite 是一个**多云管理平台（CMP）**，核心价值：

- **多云对接**：用一个控制台统一管理多家公有云/私有云的资源。
- **开箱即用**：云主机生命周期、云账单、运营分析、安全合规扫描。
- **可扩展**：多租户 + 模块化，企业可按需开发自有模块。

理解定位很重要：它不是单一应用，而是一个"平台 + 插件式业务模块"的产品形态。这决定了它的工程结构是**微服务 + 前端多包**。

## 2. 技术栈速览

### 后端（`pom.xml`）

| 维度 | 选型 |
|----|----|
| 语言/运行时 | Java 17 |
| 框架 | Spring Boot 3.1.0 + Spring Cloud 2022.0.3 |
| 服务发现/网关 | Eureka（`framework/eureka`）、Spring Cloud Gateway（`framework/gateway`） |
| 持久层 | MyBatis-Plus 3.5.3 + mybatis-plus-join |
| 数据库 | MySQL |
| 缓存/分布式锁 | Redis（Redisson 3.22） |
| 数据库版本管理 | Flyway（每个模块 `resources/db/migration`） |
| 定时任务 | Quartz（`sdk/backend/resources/quartz`） |
| 鉴权 | JWT（jjwt 0.11.5），自定义 token header `CE-TOKEN` |
| API 文档 | Swagger / OpenAPI |
| 构建 | Maven 多模块 + `flatten-plugin`（`${revision}` 统一版本） |

### 前端（`package.json`）

| 维度 | 选型 |
|----|----|
| 框架 | Vue 3 + TypeScript 4.7 |
| 构建 | Vite 3 |
| UI | Element Plus 2.3.7 + fit2cloud-ui-plus |
| 状态管理 | Pinia |
| 路由 | vue-router 4 |
| 国际化 | vue-i18n 9 |
| 图表 | Apache ECharts 5 + vue-echarts |
| HTTP | axios |
| 包管理 | Yarn 3.5.1（workspaces 单仓多包） |
| 测试 | Vitest + @vue/test-utils |
| Mock | vite-plugin-mock + mockjs |

### 基础设施

- Docker（`build-core-image.sh`、`build-core-docker-compose.sh`）
- VMware 相关依赖需先执行 `mvn initialize`（见开发指南）

## 3. 目录结构

```
CloudExplorer/
├── framework/              # 平台基座与公共能力
│   ├── eureka/             # 服务注册发现
│   ├── gateway/            # 统一网关
│   ├── management-center/  # 管理中心（用户/组织/云账号/定时任务等）
│   ├── sdk/                # 公共 SDK（前后端共享的基座能力）
│   └── provider/           # 云 provider 抽象（对接各云厂商）
├── services/               # 业务模块
│   ├── vm-service/         # 云主机管理
│   ├── finance-management/ # 云账单
│   ├── operation-analysis/ # 运营分析
│   └── security-compliance/# 安全合规
├── demo/                   # 新模块脚手架模板（DEMO-TEMPLATE）
├── doc/                    # 文档（含 开发指南.md）
├── pom.xml                 # 后端聚合 pom
└── package.json            # 前端 workspaces 根
```

每个业务模块（如 `vm-service`）内部都遵循相同结构：

```
vm-service/
├── backend/                # Spring Boot 服务
│   └── src/main/
│       ├── java/com/fit2cloud/...
│       └── resources/
│           ├── db/migration/   # Flyway 迁移脚本
│           ├── i18n/           # 国际化文案
│           └── application.yml
├── frontend/               # Vue 前端子包
│   └── src/
│       ├── views/          # 页面
│       ├── api/            # 接口封装
│       └── stores/         # Pinia
├── plugins-sdk/            # 模块自带的插件 SDK（如有）
├── vm-service.yml          # 模块描述文件（图标、菜单序号等）
└── pom.xml
```

## 4. 关键设计理念

1. **基座与业务分离**：`framework/` 提供平台能力（鉴权、用户、网关、云账号纳管），`services/` 是可替换的业务模块。
2. **每个模块自带前后端**：模块自治，前端子包由基座运行时加载。
3. **统一版本与配置**：Maven 用 `${revision}` 统一版本；运行时配置走 `/opt/cloudexplorer/conf/cloudexplorer.properties`。
4. **声明式扩展点**：模块通过 `PermissionConstants`、`MenuConstants`、`JobConstants` 三个常量类向平台"声明"自己的权限/菜单/定时任务，平台自动收集注册。

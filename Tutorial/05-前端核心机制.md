# 05 · 前端核心机制

> 前端是 Yarn workspaces 管理的 Vue 3 单仓多包工程。本章讲清楚"基座如何加载子模块、权限/菜单/路由如何串起来"。

## 1. Monorepo 结构

根 `package.json`：

```json
"workspaces": ["framework/**", "services/**"]
```

每个子包（如 `services/vm-service/frontend`）有自己的 `package.json`、`vite.config.ts`、`src/`，可独立启动与构建。根目录的 `yarn dev:vm` 实际执行 `yarn workspace vm-service run dev`。

依赖大致分两类：
- **共用依赖**（vue、element-plus、pinia、echarts）提升到根 `package.json`，避免重复安装与版本漂移。
- **模块私有依赖**写在各自 `package.json`。

## 2. 基座（ce-base / sdk frontend）

`framework/sdk/frontend`（包名通常为 `ce-base` 或 commons）是平台外壳，提供：

- **登录与鉴权**：登录、存 `CE-TOKEN`、axios 拦截器统一注入 `CE-TOKEN/CE-ROLE/CE-SOURCE` header。
- **布局**：左侧菜单、顶栏、面包屑、首页。
- **动态路由**：从后端拉取菜单声明，动态注册 vue-router 路由。
- **公共能力**：`commons` 下的 stores（user、permission、organization...）、组件、工具。

## 3. 权限指令与 Store

### 模板中

```html
<!-- 必须用带模块名的格式：[module]group:operate 或 [module]group:require+operate -->
<div v-hasPermission="'[management-center]USER:READ+CREATE'"></div>

<!-- 数组：任一命中即显示 -->
<div v-hasPermission="['[management-center]USER:READ+CREATE','[management-center]USER:READ+EDIT']"></div>
```

### TS 中

```ts
import { usePermissionStore } from "@commons/stores/modules/permission";

const permissionStore = usePermissionStore();
permissionStore.hasPermission("[vm-service]CLOUD_SERVER:STOP");
permissionStore.hasPermission(["[vm-service]CLOUD_SERVER:START","[vm-service]CLOUD_SERVER:STOP"]);
```

> 关键点：前端权限字符串**必须带模块名前缀**，因为前端要跨模块判断；后端在"当前模块内"鉴权时可省略前缀。

## 4. 路由动态生成

1. 登录后，前端调用基座接口获取当前用户可见的菜单（后端已按角色/工作空间过滤）。
2. 基座把菜单的 `path` / `componentPath` 转成 vue-router 路由记录。
3. `componentPath` 指向子模块前端的 `.vue` 文件——基座据此加载对应子模块的页面。

这意味着**新增模块的菜单会自动出现在左侧导航**，无需改基座代码。这是"声明式扩展"前端的完整闭环。

## 5. 状态管理（Pinia）

`commons/stores/modules/` 下典型 store：

- `user`：当前用户信息、角色
- `permission`：权限列表 + `hasPermission`
- `organization` / `workspace`：组织与工作空间切换
- 其他业务 store 放在各模块自己的 `src/stores/`

切换工作空间时，前端更新 `CE-SOURCE` header 并刷新权限/资源，整个多租户切换在前端完成。

## 6. HTTP 与 Mock

- **axios** 封装在 commons，统一处理：注入鉴权 header、错误码跳转登录、loading、分页约定。
- 开发期可用 `vite-plugin-mock` + `mockjs` 在 `mock/` 目录造假数据，便于脱离后端调试。

## 7. 国际化与图表

- `vue-i18n`：与后端 `i18n_xxx` key 对应，前端也维护中/英/繁体文案。
- `vue-echarts` + `echarts-liquidfill`：运营分析、账单等可视化页面大量使用，是学习 ECharts 在 Vue3 中封装的好例子。

## 8. 构建产物

`yarn build` 逐个构建子包。产物部署到 `/opt/cloudexplorer` 对应目录，由后端基座服务静态托管/按需加载。

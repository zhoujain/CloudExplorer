# 07 · 学习路径与进阶建议

## 1. 分角色学习路线

### 路线 A：我想读懂架构（适合想提升工程能力的同学）

1. 先读 [01](./01-项目概览与技术栈.md)、[03](./03-架构与模块设计.md)，建立全局视图。
2. 跑通 [02](./02-环境搭建与本地启动.md)，本地把 eureka + gateway + management-center + vm-service 起来。
3. 抓一条主线读源码：**登录 → 拿 CE-TOKEN → 访问 vm-service 列表页**，串起网关、鉴权、菜单、前端加载。
4. 读 `framework/sdk` 的公共能力，理解"基座提供什么、模块复用什么"。
5. 读 `framework/provider`，理解云厂商对接的抽象设计。

### 路线 B：我想基于它做二次开发（适合有具体业务目标的同学）

1. 快速过 [01]-[03]。
2. 精读 [04](./04-后端核心机制.md)、[05](./05-前端核心机制.md)，吃透权限/菜单/任务三件套。
3. 跟着 [06](./06-新模块开发实战.md) 用脚手架生成自己的模块，跑通闭环。
4. 复制 `vm-service` 的某个功能（如"云账号列表 + 同步"）改造成你的业务。

### 路线 C：我想学前端工程化（适合前端同学）

1. 精读根 `package.json` + Yarn workspaces 机制。
2. 读 `sdk/frontend/commons`：axios 封装、Pinia stores、权限指令 `v-hasPermission` 实现。
3. 研究基座如何**动态注册路由**（从后端菜单声明到 vue-router）。
4. 看 `operation-analysis` 如何用 vue-echarts 做可视化。

## 2. 源码阅读推荐顺序

按"由外到内、由基座到业务"：

1. `framework/eureka` — 最简单，理解服务注册。
2. `framework/gateway` — 路由 + 鉴权前置。
3. `framework/sdk/backend` — **重点**：`CeSecurityExpressionRoot`、权限/菜单/任务常量基类、JWT 处理。
4. `framework/management-center` — 平台管理全貌（用户、云账号、定时任务）。
5. `framework/provider` — 云厂商对接抽象。
6. `services/vm-service` — 第一个完整业务模块，照着学最直观。
7. `services/finance-management` / `security-compliance` — 进阶，看账单同步与合规扫描。
8. `demo/DEMO-TEMPLATE` — 模块模板，开发新模块的起点。

## 3. 值得重点学习的工程实践

| 主题 | 在哪里看 | 学什么 |
|----|----|----|
| Maven 多模块 + `${revision}` 统一版本 | 根 `pom.xml` + `flatten-plugin` | 多模块版本管理 |
| Spring Cloud 微服务 | eureka/gateway/各服务 | 服务注册、网关路由 |
| 自定义 Spring Security 表达式 | `CeSecurityExpressionRoot` | 扩展 `@PreAuthorize` |
| 声明式扩展点（Builder 模式） | 三个 `*Constants.java` | 用 Builder 描述权限/菜单/任务 |
| Flyway 数据库迁移 | 各模块 `db/migration` | 版本化 schema 演进 |
| Quartz 多类型定时任务 | `JobConstants` + `sdk/resources/quartz` | cron/间隔 + 按云账号分配 |
| Yarn workspaces monorepo | 根 `package.json` | 前端多包管理 |
| 前端动态路由 + 权限指令 | `commons` stores + `v-hasPermission` | 后端驱动的前端权限 |
| 多租户上下文 | `CE-TOKEN/CE-ROLE/CE-SOURCE` | header 化的租户切换 |
| 适配器模式对接多云 | `framework/provider` | 面向接口的云厂商抽象 |

## 4. 练手任务（由易到难）

1. **跑通本地环境**，用管理员登录，纳管一个云账号（可用 mock 或免费云）。
2. **改一个字段**：给 vm-service 的云主机列表加一列展示，走通"后端接口 → 前端表格"。
3. **加一个权限**：在 vm-service 新增一个 `CLOUD_SERVER:REBOOT` 权限，绑定到按钮，验证无权限时隐藏。
4. **生成新模块**：用 `CreateModuleUtil` 生成 `sample-module`，做一个"备忘录"CRUD，跑通声明权限/菜单 + 前后端 + Flyway。
5. **加一个定时任务**：在新模块里加一个每 5 分钟跑一次的任务，往表里写一条日志，在管理中心能看到。
6. **对接一个新云厂商**（高难度）：在 `provider` 下实现某云厂商的资源同步接口。

## 5. 学习心法

- **先跑通再读源码**：能跑起来、能点出功能，再带着"这个请求怎么走通"的问题去读，效率最高。
- **抓主线、不纠结边角**：先理解"鉴权 + 菜单 + 模块"这条主线，i18n、图表、Mock 等用到再学。
- **善用对比**：把 `vm-service` 和 `DEMO-TEMPLATE` 对比着看，能快速看出"业务模块比模板多了什么"。
- **改了就验证**：每改一处，启动验证一次，避免积累一堆改动后无法定位问题。
- **以官方文档为准**：本教程是导读，API 细节、版本差异以 <https://docs-cloudexplorer-lite.fit2cloud.com/> 与 `doc/开发指南.md` 为准。

## 6. 可能的疑问

- **Q：为什么用 Spring Cloud 而不是单体？** 多云管理需要按需启停模块、独立扩缩容（账单同步重、合规扫描重），微服务更合适。
- **Q：前端为什么不用 qiankun 等微前端框架？** 基座 + 动态路由 + 子包构建的方式更轻量，复用依赖、统一登录态，已满足需求。
- **Q：模块间能直接调用数据库吗？** 不应这样做。跨模块走 HTTP（经 Gateway），保持模块自治与可独立演进。

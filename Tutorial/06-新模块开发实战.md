# 06 · 新模块开发实战

> CE-Lite 提供了脚手架，能一键生成新模块骨架。本章带你走通"从零加一个业务模块"的完整流程。对应官方 `doc/开发指南.md` 的"新模块搭建"。

## 1. 用脚手架生成模块

打开 `demo/src/test/java/CreateModuleUtil.java`，修改以下常量：

```java
private static final String NEW_MODULE_NAME = "sample-module";          // 模块名
private static final String NEW_MODULE_DISPLAY_NAME = "样例模块";        // 中文名
private static final String NEW_MODULE_DISPLAY_NAME_TW = "樣例模塊";     // 繁体
private static final String NEW_MODULE_DISPLAY_NAME_EN = "Sample Module";// 英文
private static final long NEW_MODULE_PORT = 9021;                       // 后端端口
private static final long NEW_MODULE_FRONTEND_PORT = 5021;             // 前端 dev 端口
private static final long NEW_MODULE_MANAGEMENT_PORT = 9921;           // 服务管理端口
```

执行 `createModule()` 方法，等待完成——它会在 `services/` 下根据 `demo/DEMO-TEMPLATE` 模板生成新模块目录。

> 原理：`DEMO-TEMPLATE` 是一个最小可运行的模块样板（前后端齐全 + 三个常量类示例）。`CreateModuleUtil` 做的是模板复制 + 占位符替换（模块名、端口等）。读懂这个工具，就理解了 CE-Lite 模块的"标准结构"。

## 2. 调整模块元信息

生成后按需修改：

- `<模块名>.y`：模块描述，含图标、菜单序号等
- `backend/src/main/resources/application.yml`：端口、数据源、redis 等
- `backend/src/main/java/com/fit2cloud/ModuleApplication.java`：可重命名启动类

## 3. 实现后端三件套

### 3.1 权限（PermissionConstants.java）

声明你的模块有哪些权限组、操作，以及对哪些角色生效。例如 `sample-module` 定义 `SAMPLE:READ`、`SAMPLE:CREATE`。

### 3.2 菜单（MenuConstants.java）

声明菜单项，绑定 `componentPath` 指向前端页面，并用 `requiredPermission` 限定访问权限。

### 3.3 定时任务（JobConstants.java，按需）

如果模块需要周期性同步/扫描，声明 `JobSetting`，编写实现 `AsyncJob` 的执行器。

### 3.4 业务代码

- 实体 + Mapper（MyBatis-Plus）
- Service + Controller
- Flyway 迁移脚本 `db/migration/V1__init.sql`
- 接口上加 `@PreAuthorize("@cepc.hasCePermission('SAMPLE:READ')")`

## 4. 实现前端

`frontend/src/` 下：

- `views/`：与菜单 `componentPath` 对应的 `.vue` 页面
- `api/`：封装后端接口调用（复用 commons 的 axios）
- `stores/`：模块私有 Pinia store（如有）
- 用 `v-hasPermission` 控制按钮显隐

## 5. 注册与启动

在根 `package.json` 的 `scripts` 加上快捷命令：

```json
"dev:sample": "yarn workspace sample-module run dev",
"lint:sample": "yarn workspace sample-module run lint",
"build:sample": "yarn workspace sample-module run build-only"
```

后端把新模块注册到 Eureka（默认配置已就绪），启动后 Gateway 自动可路由。

## 6. 验证清单

- [ ] 登录后左侧出现新菜单（说明菜单声明 + 权限正确）
- [ ] 点击菜单能打开页面（说明前端 componentPath + 路由正确）
- [ ] 无权限用户看不到菜单/按钮（说明权限生效）
- [ ] 接口未带 token 返回 401（说明鉴权链路通）
- [ ] 切换工作空间，数据按租户隔离

## 7. 学习建议

第一次照着 `DEMO-TEMPLATE` 走一遍即可，重点不是"做出多复杂的功能"，而是**打通"声明权限/菜单 → 前端动态加载 → 后端鉴权 → 数据落库"的闭环**。这个闭环一旦跑通，你就真正掌握了 CE-Lite 的模块化机制。
